## Tech Debt & Warnings

### Non-Blocking

1. **`clearAllUploadWatermarks()` has no UI call site** (Phase 62) — Function implemented correctly but not yet wired to any UI action (logout, device swap). Documented as intentional in Phase 62 SUMMARY. Will be picked up in a future UI phase.

2. **DispatchSemaphore in `runHealthCheck`** (Phase 63 WR-04, skipped) — Pre-existing pattern, not introduced by v9.0. Out of scope for BLE reliability milestone.

3. **`GooseAppDelegate` APNs token race** — `pendingAPNSToken` workaround for sharedModel init race is functional but slightly fragile. Acceptable for current single-device use case.

## Milestone Goal Assessment

**Goal:** Close the critical architectural gaps identified by Ghidra RE of WHOOP v5.37.0. Phases derived from WHOOP-GOOSE-CROSS-COMPARE.md.

**Achieved:**
- ✅ Formal BLE bonding state machine (WHPBLEBondingManager parity) — Phase 61
- ✅ Per-type upload watermarks (WHPStrapLatestUploadedMetricDateKey parity) — Phase 62
- ✅ NWPathMonitor reachability gating + backoff (WHPNetworkMonitor parity) — Phase 63
- ✅ Swift-side HR spike filter (WHPHeartRateDataSanitizer parity) — Phase 64
- ✅ Generic BLE state machine type (WHPStateMachine parity) — Phase 65
- ⛔ Cap sense GATT UUID + on-wrist detection (WHPWhoopStrapOnWrist parity) — DEFERRED

**Core value delivered:** The band's BLE reliability and upload correctness are now significantly improved. The upload pipeline is idempotent (watermarks), network-aware (reachability gate), and HR data is sanitized before reaching algorithms.
